Bare-bones multi-year PyPSA pathway model

Implementation spec for a one-zone power-only pilot covering 2030, 2040 and 2050. Linear capacity expansion and dispatch with wind, solar, gas turbine and load shedding. Solver: HiGHS. Development style: red/green TDD.

1. Scope

Build the smallest useful PyPSA model for testing multi-investment-period logic. The model is intentionally not policy-grade: no existing fleet, no CO₂ accounting, no storage, no transmission, no unit commitment.

2. Model formulation

Investment periods 2030, 2040, 2050. Period weights are configurable.
Temporal resolution 8,760 hourly snapshots per investment period.
Technologies Wind, solar, gas turbine, load shedding.

Decision variables

VariableDescription
CapacityEndogenous capacity for wind, solar and gas turbine by build year.
DispatchHourly generation from wind, solar, gas turbine and load shedding.

Objective

Minimise discounted system cost: investment cost + fixed O&M + gas variable generation cost + load shedding penalty.

Constraints

ConstraintPurpose
Hourly power balanceDemand must be met in every hour and period.
RES availabilityWind and solar dispatch limited by hourly capacity factors.
Capacity potentialsMaximum buildable wind and solar capacity.
Lifetime logicCapacity built in earlier periods remains active if lifetime allows.
Load sheddingHigh-cost feasibility backstop.
Important: without CO₂ cost/caps, gas may dominate if assumed gas marginal cost is low. That is acceptable for debugging but not for a transition-to-zero-emissions result.

8. Implementation sketch

Build network

def build_network(cfg, data):
    import pypsa
    import pandas as pd

    network = pypsa.Network()

    snapshots = []
    for period in cfg.investment_periods:
        hours = pd.date_range(f"{period}-01-01", periods=8760, freq="h")
        snapshots.extend([(period, ts) for ts in hours])

    network.snapshots = pd.MultiIndex.from_tuples(
        snapshots,
        names=["period", "timestep"],
    )
    network.investment_periods = pd.Index(cfg.investment_periods)
    network.investment_period_weightings["objective"] = pd.Series(cfg.period_weights)

    network.add("Bus", "electricity")

    # Add one load time series indexed by the multi-period snapshots.
    network.add(
        "Load",
        "demand",
        bus="electricity",
        p_set=data.demand_series,
    )

    # Extendable renewable generators.
    network.add(
        "Generator",
        "wind",
        bus="electricity",
        carrier="wind",
        p_nom_extendable=True,
        p_nom_max=cfg.tech.wind.max_capacity_mw,
        capital_cost=cfg.tech.wind.annualised_cost_by_period,
        p_max_pu=data.wind_cf_series,
        lifetime=cfg.tech.wind.lifetime_years,
        build_year=2030,
    )

    network.add(
        "Generator",
        "solar",
        bus="electricity",
        carrier="solar",
        p_nom_extendable=True,
        p_nom_max=cfg.tech.solar.max_capacity_mw,
        capital_cost=cfg.tech.solar.annualised_cost_by_period,
        p_max_pu=data.solar_cf_series,
        lifetime=cfg.tech.solar.lifetime_years,
        build_year=2030,
    )

    network.add(
        "Generator",
        "gas_turbine",
        bus="electricity",
        carrier="gas",
        p_nom_extendable=True,
        p_nom_max=cfg.tech.gas_turbine.max_capacity_mw,
        capital_cost=cfg.tech.gas_turbine.annualised_cost_by_period,
        marginal_cost=cfg.tech.gas_turbine.variable_cost_by_period,
        lifetime=cfg.tech.gas_turbine.lifetime_years,
        build_year=2030,
    )

    network.add(
        "Generator",
        "load_shedding",
        bus="electricity",
        carrier="load_shedding",
        p_nom=cfg.tech.load_shedding.max_capacity_mw,
        marginal_cost=cfg.tech.load_shedding.variable_cost_eur_per_mwh,
    )

    return network
This is only a sketch. In real PyPSA multi-investment models, period-specific investment options are often represented as separate assets per build year, e.g. wind_2030, wind_2040, wind_2050, each with its own build_year, cost and lifetime. Use that pattern if one generator with period-varying costs becomes awkward.

9. Preferred simple asset pattern

For clarity, use one generator per technology-build-year: 

wind_2030, wind_2040, wind_2050
solar_2030, solar_2040, solar_2050
gas_turbine_2030, gas_turbine_2040, gas_turbine_2050

This makes costs, build years and lifetimes explicit. It also avoids ambiguity about whether a single extendable asset can invest repeatedly at changing costs.

12. Keep out of v0

Do not add these until the pilot solves and produces interpretable results:

FeatureReason to defer
StorageAdds time-coupled state variables and complicates interpretation.
CO₂ constraintsImportant later, but not needed for debugging pathway mechanics.
Multiple zonesTransmission and spatial data will distract from core logic.
Unit commitmentTurns a simple LP into a much harder MILP.
Endogenous learningCan require nonlinear or approximate formulations.

13. Main caveats

  • Perfect foresight: the model knows future costs and demand when investing in 2030.
  • Sparse periods: 2030, 2040 and 2050 may create lumpy build-outs.
  • No existing fleet: useful for testing, unrealistic for transition analysis.
  • No CO₂ accounting: results should not be described as decarbonisation pathways.
  • Synthetic data: fine for TDD, not for conclusions.

Recommended next step after v0: add existing fleet and CO₂ caps before interpreting results as a transition pathway.
